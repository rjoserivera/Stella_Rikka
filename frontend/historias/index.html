💡 Guía de Uso: Historias de Usuario

✚
1. Agregar una Historia Haz clic en "+ Agregar Historia". Se abrirá un formulario completo donde podrás llenar el ID, título, épica y la narrativa estándar: "Como [rol], quiero [acción], para [beneficio]".
📋
2. Criterios de Aceptación (Gherkin) Dentro del formulario de edición podrás añadir múltiples criterios con el formato Dado/Cuando/Entonces. Cada criterio se puede añadir o eliminar individualmente para definir exactamente cuándo la historia está "terminada".
🔄
3. Cambiar Estado desde la Lista Haz clic directamente en el estado de cualquier historia para actualizarlo al instante. El color de la fila cambiará para reflejar su estado actual: Backlog (gris), En Sprint (azul), En Desarrollo (naranja), En Testing (morado) y Terminada (verde).
🔗
4. Trazabilidad al Requerimiento Cada historia puede vincularse a su requerimiento de origen (ej: RF-001) para mantener la trazabilidad completa entre análisis y diseño ágil.
🖼️
5. Exportar y Guardar Exportar PNG genera una imagen limpia de toda la tabla (sin botones). Guardar como... descarga un JSON con todos los datos para respaldo o para importar en otra sesión.
