❓ ¿Cómo usar la Matriz MoSCoW?

La Matriz MoSCoW es una técnica de priorización utilizada en desarrollo de software para acordar con los interesados (stakeholders) qué requerimientos son esenciales y cuáles pueden posponerse.

🔴 Obligatorio (Must Have)

Requerimientos vitales. Si falta alguno de estos, el producto no sirve o no se puede lanzar. El proyecto fracasará si no se cumplen.

🟠 Importante (Should Have)

Requerimientos de alta prioridad pero no críticos para el lanzamiento. Si se omiten, el producto funcionará, pero el valor disminuirá.

🔵 Deseable (Could Have)

Requerimientos "nice-to-have". Solo se incluirán si sobra tiempo y presupuesto. Mejoran la experiencia pero no son esenciales.

⬛ Descartado (Won't Have)

Requerimientos acordados para ser excluidos en esta fase. Ayudan a mantener el alcance claro y evitar que el proyecto crezca sin control.

🖱️ Cómo usar esta aplicación

  • Usa el botón "+" en cada columna o el botón flotante inferior para agregar funcionalidades.
  • Arrastra y suelta (Drag & Drop) las tarjetas entre columnas para re-priorizarlas fácilmente.
  • Haz clic en una tarjeta para editar su título, descripción, esfuerzo o prioridad.
  • Revisa el "📊 Resumen" arriba. Lo ideal es que los "Must Have" no superen el 60% del esfuerzo total.
